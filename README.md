# Logistics & Shipping Linting and Best Practices Enforcer VS Code Extension

> A VS Code extension that lints and enforces best practices for logistics and shipping code, including tracking numbers, address validation, carrier APIs, and documentation. It reduces errors by flagging invalid formats, deprecated integrations, and security vulnerabilities in real-time. Ideal for developers working with shipping libraries, e-commerce platforms, and supply chain systems.

## Installation

### From VS Code Marketplace
1. Open VS Code
2. Press `Ctrl+P` (or `Cmd+P` on Mac)
3. Type `ext install digitalforge.13520e21`
4. Press Enter

### From VSIX file
1. Download the `.vsix` file
2. In VS Code: `Extensions > ··· > Install from VSIX`
3. Select the downloaded file

## Usage

Press `Ctrl+Shift+P` and search for `Logistics & Shipping Linting and Best Practices Enforcer VS Code Extension` to see all available commands.

**Keyboard shortcut:** `Ctrl+Shift+Alt+A` (or `Cmd+Shift+Alt+A` on Mac)

## Settings

| Setting | Default | Description |
|---------|---------|-------------|
| `13520e21.enabled` | `true` | Enable/disable the extension |
| `13520e21.autoRun` | `false` | Auto-run on file open |
| `13520e21.logLevel` | `"info"` | Log verbosity |
